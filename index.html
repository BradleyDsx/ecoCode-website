Le numérique nous passionne et nous considérons qu’il est impératif de réduire son impact environnemental et social afin de le rendre plus durable.

ecoCode est une initiative qui prend racine dans la force du collectif
ecoCode rassemble différents types d'acteurs qui oeuvrent ensemble pour rendre le numérique plus responsable : recherche, entreprises, personnes en poste sur leur temps de travail, personnes indépendantes sur leur temps libre...
Les membres contributeurs du collectif conceptualisent et implémentent des règles permettant de repérer les défauts de qualité du code source d’un projet pouvant avoir un impact écologique négatif. Le tout est mis à disposition en open source.

Le processus de création d'une règle ecoCode

identification
Nous identifions des patterns de code qui pourraient être améliorés (code smells)
production
Les membres contributeurs produisent la règle permettant de réduire l’impact du code
expérimentation
La règle est testée en comparaison avec le code smell
utilisation
Utilisation de la règle par des équipes techniques dans leur organisation
intégration
La règle est publiée dans le référentiel Open Source d’ecoCode
validation
Une équipe expérimentée étudie et valide la nouvelle règle
